# 和みの大人夏合宿 in 淡路島 — 共有LP 実装仕様書

## 1. 技術スタック / 制約

### 禁止事項
- `localStorage` / `sessionStorage` にアプリデータを保存しない（共有されないため）。データは必ず `window.storage`。
- HTMLの `<form>` タグでの送信は使わない。`onClick` / `onChange` で処理。
- 外部画像が読めない環境でも崩れないよう、**全写真にフォールバックのグラデーション背景**を敷く（§8）。

## 6. データモデル（`window.storage`・全員共有）

各キーは `window.storage.set(key, JSON, /*shared=*/true)` で保存、`get(key, true)` で取得。**shared=true 必須**（全員同期のため）。

| キー | 型 | 用途 |
|---|---|---|
| `pay` | `{ [名前]: { cash:bool, car:bool } }` | 集金チェック |
| `events` | `{ [key]: string[] }` | 各イベント参加者。key= sup/sauna/fishing/bbq/fireworks/cards |
| `shared` | `[{id,text,done,by}]` | 共有の持ち物 |
| `konan` | `[{id,text,done,by}]` | 買い出し（コーナン） |
| `ropia` | `[{id,text,done,by}]` | 買い出し（ロピア） |
| `pack` | `{ [名前]: [{id,text,done}] }` | 各自の持ち物 |
| `board` | `[{id,by,text}]` | 連絡ボード（先頭が最新） |

- `id` は `Date.now().toString(36)+乱数` で一意化。
- 書き込み後は保存インジケータ（「保存しました ✓」を一瞬表示）を出す。
- 保存・取得は必ず `try/catch`。取得失敗時は初期値でフォールバック。
- 全ての追加/削除/チェック/参加トグルの後、**再レンダリング**して即時反映。

### 保存関数の雛形
```js
const store = {
  async get(k,d){ try{ const r=await window.storage.get(k,true); return r?JSON.parse(r.value):d; }catch(e){ return d; } },
  async set(k,v){ try{ await window.storage.set(k,JSON.stringify(v),true); toast(); }catch(e){ console.error(e); } }
};
```

## 8. 写真とフォールバック（重要）

- 写真は外部URL（例: Unsplash）を `<img>` で読み込む。**読み込み成功時のみ表示**し、失敗時はテーマ色のグラデーション背景を残す二重構造にする。
- 各写真枠に対応する `FALLBACK` グラデを用意（例: hero=海ブルー、bbq=テラコッタ、fireworks=紫、stars=紺）。
- 実装: `new Image()` で先読みし、`onload` で `<img>.src` をセット＋`.loaded` クラス付与（`opacity:0→1`）。`onerror` は何もしない（グラデ維持）。
- **確実に表示したい写真がある場合**は、その画像をBase64でHTMLに埋め込む方式に差し替え可能（幹事が写真を用意したら対応）。

```js
function loadImages(){
  document.querySelectorAll('[data-img]').forEach(el=>{
    const src = IMG[el.dataset.img]; if(!src) return;
    const probe = new Image();
    probe.onload = ()=>{ el.src=src; el.classList.add('loaded'); };
    probe.src = src;
  });
}
```

## 9. 環境の注意（アーティファクト外で使う場合）

- `window.storage` はClaudeアーティファクト専用。もしXserver等の独自ホストに載せる場合は、`window.storage` を **Firebase Realtime Database / Firestore もしくは Supabase** のラッパーに差し替える（同じ `get/set` インターフェースを実装）。それによりデータの全員同期を維持できる。
- その場合、公開URLは独自ドメインで発行でき、閲覧者のアカウントも不要にできる。
